# Материалы для отчета

## 1. Тема проекта

Тема лабораторного проекта: «Разработка сверточной нейронной сети для классификации типа транспортного средства по изображению».

## 2. Цель работы

Целью работы является получение практических навыков проектирования, программной реализации, обучения и оценки сверточной нейронной сети для решения задачи многоклассовой классификации изображений. В рамках работы используется язык программирования Python и библиотека TensorFlow/Keras.

## 3. Постановка задачи

Необходимо разработать программную систему, которая принимает на вход изображение транспортного средства и определяет его тип. Задача относится к классу задач компьютерного зрения и является задачей многоклассовой классификации, поскольку каждое изображение должно быть отнесено ровно к одному из нескольких заранее заданных классов.

Ожидаемые классы датасета включают: Big Truck, City Car, Multi Purpose Vehicle, Sedan, Sport Utility Vehicle, Truck и Van. Фактический список классов в программе определяется автоматически по названиям папок с изображениями.

## 4. Техническое задание

В ходе выполнения проекта требуется:

- подготовить структуру Python-проекта;
- загрузить и проверить локально размещенный датасет Kaggle;
- автоматически определить классы по директориям;
- разделить данные на обучающую, валидационную и тестовую выборки;
- выполнить изменение размера изображений и нормализацию пикселей;
- применить аугментацию к обучающей выборке;
- разработать архитектуру сверточной нейронной сети;
- обучить модель с использованием TensorFlow/Keras;
- сохранить обученную модель и словарь классов;
- построить графики accuracy и loss;
- оценить качество модели на тестовой выборке;
- построить classification report и confusion matrix;
- реализовать интерфейс на Flask для загрузки изображения и вывода результата классификации.

## 5. Описание исходных данных

Исходные данные представляют собой набор изображений транспортных средств, разделенных по папкам в соответствии с классами. Каждая подпапка соответствует отдельному типу транспортного средства. Такой формат удобен для задач классификации изображений, так как метка класса может быть получена непосредственно из имени директории.

Изображения могут иметь различное разрешение, ориентацию и условия съемки. Для корректной подачи в нейронную сеть все изображения приводятся к единому размеру `224x224` пикселя и преобразуются в трехканальный RGB-формат.

## 6. Источник исходных данных

В проекте используется открытый датасет Kaggle `Vehicle Images Dataset`.

Источник: https://www.kaggle.com/datasets/lyensoetanto/vehicle-images-dataset

Kaggle slug: `lyensoetanto/vehicle-images-dataset`.

Датасет может быть скачан вручную через веб-интерфейс Kaggle или с использованием Kaggle API командой:

```bash
kaggle datasets download -d lyensoetanto/vehicle-images-dataset
```

После скачивания архив распаковывается в директорию `data/raw/`.

## 7. Обоснование выбора CNN

Для решения поставленной задачи выбрана сверточная нейронная сеть, поскольку она хорошо подходит для обработки изображений. Изображение обладает пространственной структурой: соседние пиксели связаны между собой и образуют локальные визуальные признаки. Полносвязная нейронная сеть не учитывает такую локальную структуру напрямую и требует значительно большего числа параметров.

Сверточные слои позволяют выделять локальные признаки изображения: края, линии, контуры, текстуры, формы кузова, колеса, окна и другие детали транспортных средств. На более глубоких уровнях сети отдельные локальные признаки объединяются в более сложные визуальные представления, которые помогают отличать один тип транспортного средства от другого.

Операции pooling уменьшают пространственный размер признаковых карт, снижают вычислительную сложность и делают модель менее чувствительной к небольшим смещениям объектов на изображении. Выходной слой с функцией активации softmax используется для получения распределения вероятностей по всем классам.

## 8. Описание архитектуры нейросети

В проекте используется одна модель на базе архитектуры `EfficientNetB0`. Данная архитектура является сверточной нейронной сетью и применяется в режиме transfer learning: сверточный backbone предварительно обучен на датасете ImageNet, а для текущей задачи добавлена новая классификационная голова.

- входной слой для изображения размера `224x224x3`;
- преобразование нормализованных пикселей обратно в диапазон `0..255`, необходимый для EfficientNetB0;
- предобученный сверточный backbone `EfficientNetB0`;
- слой `GlobalAveragePooling2D`;
- слой `BatchNormalization`;
- слой `Dropout`;
- полносвязный слой `Dense(256, activation='relu')`;
- дополнительный слой `Dropout`;
- выходной слой `Dense(num_classes, activation='softmax')`.

Использование EfficientNetB0 позволяет применить признаки, заранее изученные на большом наборе изображений. Это повышает качество классификации по сравнению с обучением небольшой CNN с нуля, особенно если предметный датасет имеет ограниченный размер. Функция активации ReLU используется в полносвязном слое классификационной головы. BatchNormalization стабилизирует распределение активаций, а Dropout применяется для уменьшения риска переобучения.

## 9. Описание подготовки данных

Подготовка данных включает несколько этапов. Сначала программа проверяет наличие папки `data/raw/` и ищет директорию, содержащую подпапки классов. Затем автоматически формируется список классов и подсчитывается количество изображений в каждом классе.

После этого изображения разделяются на три выборки: обучающую, валидационную и тестовую. В проекте используется соотношение: 70% для обучения, 15% для валидации и 15% для тестирования. При возможности применяется стратифицированное разбиение, чтобы сохранить приблизительное соотношение классов в каждой выборке.

Каждое изображение загружается, декодируется в RGB-формат, изменяется до размера `224x224` и нормализуется делением значений пикселей на 255. Для обучающей выборки дополнительно используется аугментация: горизонтальное отражение, небольшой поворот, масштабирование и изменение контраста. Аугментация повышает разнообразие обучающих данных и снижает вероятность переобучения.

## 10. Описание параметров обучения

Основные параметры обучения:

- размер изображения: `224x224`;
- размер batch: `32`;
- обучение классификационной головы: `8` эпох;
- fine-tuning верхних слоев EfficientNetB0: `20` эпох;
- количество размораживаемых верхних слоев backbone: `40`;
- оптимизатор: `Adam`;
- функция потерь: `sparse_categorical_crossentropy`;
- метрика качества: `accuracy`;
- режим весов классов по умолчанию: `none`;
- размер тестовой выборки: `15%`;
- размер валидационной выборки: `15%`.

Функция потерь `sparse_categorical_crossentropy` выбрана потому, что метки классов представлены целыми числами. Оптимизатор Adam сочетает идеи адаптивной скорости обучения и метода моментов, что делает его удобным базовым выбором для обучения нейронных сетей. На первом этапе используется learning rate `1e-3`, а на этапе fine-tuning используется меньший learning rate `1e-5`, чтобы аккуратно адаптировать предобученные веса к изображениям транспортных средств.

## 11. Описание процесса обучения

Обучение выполняется скриптом `src/train_model.py`. После загрузки и разделения данных создается `tf.data` pipeline, который обеспечивает эффективную загрузку, предварительную обработку, пакетирование и предзагрузку данных.

Процесс обучения состоит из двух этапов. На первом этапе EfficientNetB0 заморожен и используется как извлекатель признаков; обучаются только новые слои классификационной головы. На втором этапе размораживается верхняя часть EfficientNetB0 и выполняется fine-tuning, позволяющий адаптировать высокоуровневые признаки к конкретным классам транспортных средств.

Во время обучения используются callback-функции и вспомогательные механизмы:

- сохранение лучшей модели по значению validation accuracy;
- `EarlyStopping` останавливает обучение при отсутствии улучшения validation loss;
- `ReduceLROnPlateau` уменьшает скорость обучения, если validation loss перестает улучшаться.

Для учета дисбаланса классов предусмотрены опциональные class weights. По умолчанию они отключены, поскольку при оптимизации общей accuracy слишком сильное взвешивание малых классов может ухудшать качество на крупных классах. При необходимости может использоваться мягкий режим `sqrt` или более сильный режим `balanced`.

По завершении обучения одна итоговая модель сохраняется в файл `models/vehicle_type_cnn.keras`, а словарь соответствия классов и индексов сохраняется в `models/class_indices.json`. Графики accuracy и loss сохраняются в `reports/figures/training_history.png`.

## 12. Анализ результатов

Качество модели оценивается на тестовой выборке, которая не используется при обучении и подборе параметров. Основной метрикой является accuracy, показывающая долю правильно классифицированных изображений.

Дополнительно строится classification report, включающий precision, recall и f1-score для каждого класса. Precision показывает, насколько надежны предсказания определенного класса. Recall показывает, какая доля объектов данного класса была правильно обнаружена. F1-score является гармоническим средним precision и recall.

Confusion matrix позволяет визуально проанализировать, какие классы модель путает между собой. Для данной предметной области возможны ошибки между внешне похожими типами транспорта, например между Sedan и City Car или между Truck и Big Truck.

## 13. Описание пользовательского интерфейса

Для практического использования обученной модели реализован простой веб-интерфейс на Flask. Пользователь открывает локальную страницу в браузере, загружает изображение транспортного средства и получает результат классификации.

Интерфейс отображает загруженное изображение, предсказанный класс и вероятности по всем классам в процентах. Это позволяет не только увидеть итоговое решение модели, но и оценить степень уверенности классификатора.

Запуск интерфейса выполняется командой:

```bash
python app/app.py
```

После запуска приложение доступно по адресу `http://127.0.0.1:5000/`.

## 14. Выводы

В результате выполнения проекта разработана программная система для классификации типа транспортного средства по изображению. В ходе работы были выполнены основные этапы решения задачи компьютерного зрения: подготовка датасета, построение архитектуры сверточной нейронной сети, обучение модели, оценка качества и разработка пользовательского интерфейса.

Использование CNN является обоснованным, поскольку сверточные слои эффективно извлекают пространственные признаки изображений. Применение transfer learning на базе EfficientNetB0 позволяет повысить качество распознавания за счет использования предварительно обученных визуальных признаков. Нормализация, аугментация, BatchNormalization, Dropout, опциональные class weights и callback-функции делают процесс обучения более устойчивым и снижают риск переобучения.

Полученный проект может быть использован как основа для учебной защиты, дальнейших экспериментов с архитектурами нейронных сетей и улучшения качества классификации за счет расширения датасета, настройки гиперпараметров или применения transfer learning.
